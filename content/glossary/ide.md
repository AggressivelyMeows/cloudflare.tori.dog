An **IDE** (Integrated Development Environment) is an app on your computer where you write, run, and debug code.

An IDE combines a code editor, error checker, terminal, and debugger in one application. Most IDEs understand the language you're writing in and can catch mistakes as you type.

> **Think of it as:** a word processor for code that also checks your spelling, explains your errors, and runs your program.

## The Words You'll See Most

**Editor**
The part of the IDE where you actually type your code. Most editors highlight different parts of your code in different colors (called *syntax highlighting*) to make it easier to read.

**Terminal**
A built-in command line inside the IDE. You can run tools like `npm`, `pnpm`, or `wrangler` without leaving the app.

**Extension / Plugin**
A small add-on that adds support for a new language, a linter, or a theme. VS Code has thousands of free extensions.

**Debugger**
A tool that lets you pause your running program, look at what's happening inside it, and find out where something went wrong.

**IntelliSense / Autocomplete**
When the IDE suggests how to finish what you're typing. It can fill in function names, show what arguments a function expects, and display inline documentation.

## Common IDEs

**VS Code** (Visual Studio Code) is the most popular free IDE for web and JavaScript development.

**WebStorm** is a paid IDE from JetBrains with deeper JavaScript and TypeScript support built in.

**Cursor** is a newer AI-focused editor built on top of VS Code.

## When Would I Need an IDE?

Any time you're writing code. A basic text editor works, but an IDE catches typos, autocompletes your code, and runs your project without switching tools.
