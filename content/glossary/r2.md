**R2** is a place to store files on Cloudflare, photos, videos, PDFs, anything.

Think of it like a giant online folder that lives on the internet. You upload files to it, and you can grab them back whenever you want, from anywhere.

> **The big perk:** Most storage services charge you a fee every time you *download* your files. R2 doesn't. That can save a lot of money if people access your files often.

## The 4 Words You'll See Most

**Bucket**
A container for your files. Think of it as one big folder. You might have a bucket called `my-photos` and another called `backups`.

**Object**
A single file stored in a bucket, like one image or one video. (Yes, "object" just means "file" here.)

**Key**
The name of a file inside a bucket, including any folder path. For example, `2026/summer/beach.jpg`.

**Egress**
A fancy word for "data leaving Cloudflare" (like people downloading your files). With R2, this is **free**, which is the main reason people switch to it.

## A Tiny Example

Here's how you might save and read a file using a Worker:

```js
export default {
  async fetch(request, env) {
    // save a file into the bucket
    await env.MY_BUCKET.put("hello.txt", "Hi there!");

    // read it back
    let file = await env.MY_BUCKET.get("hello.txt");

    return new Response(await file.text());
  }
};
```

## When Would I Use R2?

- Storing **user uploads** like profile pictures
- Serving **images or videos** for a website
- Keeping **backups** of your data
- Hosting **large files** people download a lot
